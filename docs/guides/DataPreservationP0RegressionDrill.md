# Phase 1.11.7 P0 数据保全回归与事故演练

## 目标

Phase 1.11.7 不新增业务能力，而是把 Phase 1.11 已落地的数据保全能力转成可重复执行的回归清单和事故演练流程。每次发布前、同步相关改动后、数据恢复相关改动后，都应按本指南验证“有效用户首页不会被静默覆盖，且覆盖前有可预览、可恢复版本”。

核心验证目标：

- 默认页、空白页和未编辑模板页不会污染本地或云端历史。
- 用户正常编辑首页和用户编辑后的模板页会被识别为有效用户首页。
- 所有危险覆盖入口在覆盖前保护当前有效用户首页。
- 系统态首页不会自动上传覆盖云端有效首页。
- 本地覆盖云端前会先保护当前云端版本。
- 本地历史和账号托管云端历史都可以完整预览并恢复到本机。
- 数据恢复后默认暂停同步，不静默覆盖云端。
- 数据包导出和本地审计不泄露同步码、账号托管 secret 或登录 session。

## 适用时机

必须执行：

- 修改 `HomeDocumentV2`、模板、导入、恢复、同步、首页空间、账号托管或数据恢复中心相关代码后。
- 修改 Supabase RPC、RLS、Storage policy 或账号托管历史版本迁移后。
- GitHub Pages 部署前的 P0 发布检查。
- 线上出现首页清空、误覆盖、同步冲突异常、恢复失败或云端历史缺失时。

建议执行：

- 主题、Banner/背景、组件数据结构变更后。
- 大批量导入、模板库、组件默认配置扩展后。
- 正式域名、Auth redirect 或缓存域隔离变更后。

## 环境准备

至少准备：

- 浏览器 A：当前主浏览器，用于登录账号和账号托管空间。
- 浏览器 B：另一个浏览器、无痕窗口或同浏览器独立 profile，用于模拟第二设备。
- 测试账号：可登录 Supabase Auth，并能创建账号托管首页空间。
- 一个普通同步码空间：用于验证密文同步边界和冲突处理。
- 一个账号托管空间：用于验证云端历史版本和恢复到本机。

推荐测试数据：

- 至少 3 个分组。
- 至少 8 个网站，包含未分组和分组内网站。
- 至少 2 个组件，例如 Todo 和月历。
- 一个非默认主题。
- Banner 或背景至少设置一种；如无法使用 Storage，可使用外链图片。
- 修改过的模板首页一份，用于验证编辑后模板属于有效用户数据。

## 发布前自动检查

每次演练前先运行：

```powershell
npm run typecheck
npm run lint
npm run build
git diff --check
```

如涉及 Supabase 迁移或权限，额外执行：

```text
supabase/checks/014_cloud_home_snapshots_verify.sql
supabase/checks/015_account_managed_recovery_model_verify.sql
```

验收要求：

- 自动检查全部通过。
- `014` 验证账号托管云端历史表、审计表、RLS、grant、RPC 和旧同步码兼容。
- `015` 验证账号托管恢复模型的权限边界，并确认 `anon` / `PUBLIC` 不能直接访问敏感表。

## 调试观察点

浏览器控制台常用只读检查：

```js
Object.entries(localStorage).filter(([key]) => key.startsWith("homepage:"));
```

查看本地历史数量：

```js
JSON.parse(localStorage.getItem("homepage:local-snapshots:v1") || "[]").length;
```

查看当前文档 revision 和更新时间：

```js
const current = JSON.parse(localStorage.getItem("homepage:document:v2") || "null");
({ revision: current?.revision, updatedAt: current?.updatedAt, groups: current?.groups?.length, widgets: current?.widgets?.length });
```

查看最近本地审计：

```js
JSON.parse(localStorage.getItem("homepage:audit-log:v1") || "[]").slice(0, 10);
```

注意：不要在生产用户浏览器中直接修改 localStorage。需要模拟损坏数据时，只在测试 profile 中操作，并提前导出当前数据包。

| ID | 场景 | 前置条件 | 操作 | 通过标准 |
|---|---|---|---|---|
| P0-117-01 | 默认页保护 | 浏览器为系统默认首页 | 恢复默认、应用模板或导入覆盖 | 不生成有效本地快照；操作可继续；不会自动上传系统态覆盖云端 |
| P0-117-02 | 空白页保护 | 当前为空白首页 | 执行 JSON 导入或数据包恢复 | 覆盖前不生成系统态快照；导入成功后如用户确认保留，后续视为有效用户首页 |
| P0-117-03 | 未编辑模板保护 | 当前为刚应用但未编辑的模板页 | 恢复默认或切换模板 | 不生成有效快照；不会自动同步覆盖云端 |
| P0-117-04 | 编辑后模板保护 | 应用模板后新增或修改一个网站 | 恢复默认 | 恢复默认前生成本地快照；恢复后如有同步绑定，进入暂停状态 |
| P0-117-05 | 恢复默认 | 当前为有效用户首页 | 点击恢复默认并确认 | 覆盖前生成本地快照；数据恢复中心可预览旧版本；云端不被静默覆盖 |
| P0-117-06 | 应用空白/模板 | 当前为有效用户首页 | 首页欢迎区或设置页应用空白页/模板 | 覆盖前生成本地快照；系统态目标不自动上传 |
| P0-117-07 | JSON 导入 | 当前为有效用户首页 | 高级操作导入 JSON 并确认覆盖 | 覆盖前生成本地快照；导入后文档 revision 增加；审计有导入记录 |
| P0-117-08 | 数据包恢复 | 当前为有效用户首页且有测试数据包 | 从数据包恢复并确认 | 覆盖前生成本地快照；不恢复 secret/session；如有同步绑定，恢复后暂停同步 |
| P0-117-09 | 书签/URL 导入 | 当前为有效用户首页 | 导入书签 HTML 或 URL 列表并确认写入 | 写入前生成本地快照；导入后可撤销；撤销前也生成保护快照 |
| P0-117-10 | 本地历史恢复 | 数据恢复中心存在本地版本 | 预览并恢复任一版本 | 预览可滚动并完整展示分组、网站、组件、主题和图片状态；恢复前再生成当前首页快照 |
| P0-117-11 | 本地历史去重 | 当前首页内容与最新快照一致 | 连续执行同内容覆盖 | 不生成重复内容指纹快照；操作可继续 |
| P0-117-12 | 本地历史上限 | 测试 profile 中构造超过 30 个本地快照 | 连续触发危险覆盖 | 只保留最近 30 个本地快照 |
| P0-117-13 | 损坏本地历史 | 测试 profile 中将 `homepage:local-snapshots:v1` 改成非法 JSON | 打开设置页和数据恢复中心 | 页面不崩溃；恢复中心显示空态或安全提示；主流程可继续 |
| P0-117-14 | 同步码绑定覆盖本地 | 当前有效本地首页，准备输入另一个同步码 | 绑定同步码并拉取云端 | 覆盖本地前生成本地快照；失败时取消覆盖 |
| P0-117-15 | 手动拉取云端 | 当前有效本地首页且已绑定同步空间 | 点击拉取云端并确认 | 覆盖本地前生成本地快照；云端内容写回本机后不丢失旧本地版本 |
| P0-117-16 | 本地覆盖云端 | 云端已有有效用户首页 | 点击上传本地或冲突中选择本地覆盖云端 | 强确认；先拉取并保存当前云端版本到本地恢复中心；保护失败则不覆盖云端 |
| P0-117-17 | 系统态自动上传阻断 | 已绑定同步空间后恢复默认或应用空白页 | 等待自动同步周期或触发同步检查 | 不自动上传系统态；同步状态进入暂停；提示出现在账号当前首页或对应同步上下文 |
| P0-117-18 | 同步冲突使用云端 | 浏览器 A/B 分别修改同一空间 | 在冲突提示中选择云端版本 | 覆盖本地前生成本地快照；本机恢复为云端版本 |
| P0-117-19 | 同步冲突使用本地 | 浏览器 A/B 分别修改同一空间 | 在冲突提示中选择本地覆盖云端 | 强确认；先保护当前云端版本；账号托管空间成功后写云端审计 |
| P0-117-20 | 账号托管云端历史 | 当前空间为账号托管，上传有效用户首页 | 等待自动上传或手动上传 | 云端历史列表出现新版本；可完整预览；重复内容不新增重复快照 |
| P0-117-21 | 云端历史恢复到本机 | 账号托管空间存在云端历史 | 在恢复中心预览并恢复云端版本 | 恢复前保护当前本地首页；恢复后同步暂停；云端 revision 不变化 |
| P0-117-22 | 普通同步码隐私边界 | 当前为空普通同步码空间 | 打开数据恢复中心 | 不显示可预览的云端明文历史；同步码空间仍只依赖本地历史和密文云端 |
| P0-117-23 | 数据包导出脱敏 | 当前有账号托管空间和同步码绑定 | 导出数据包并搜索敏感字段 | 不包含 `accessToken`、`encryptionKey`、`access_token`、`encryption_key`、Supabase session 或云端历史 `document_json` |
| P0-117-24 | 多标签同步协调 | 同浏览器打开两个标签页并绑定同一空间 | 同时触发同步检查、拉取或上传 | 请求被本地锁串行化；不会出现两个标签同时强制覆盖 |
| P0-117-25 | 多设备误覆盖演练 | 浏览器 A/B 都有有效用户首页并分别修改 | 分别执行拉取、上传、冲突选择 | 每个覆盖方向都有可恢复快照；云端和本地恢复路径清楚 |

## 标准事故演练流程

### 1. 冻结现场

- 停止继续编辑当前首页。
- 不要立即点击“上传本地”或“拉取云端”。
- 保留当前浏览器标签页，避免丢失内存状态。
- 在设置页导出数据包，作为事故现场副本。

### 2. 记录上下文

记录：

- 是否已登录。
- 当前首页空间名称和 access mode。
- 当前 revision、更新时间和同步状态。
- 最近一次操作：恢复默认、导入、空间切换、云端拉取、上传、冲突解决等。
- 数据恢复中心本地历史数量和云端历史数量。
- 本地审计最近 10 条。

### 3. 判断恢复来源

优先顺序：

1. 当前浏览器数据恢复中心的本地历史版本。
2. 账号托管空间的数据恢复中心云端历史版本。
3. 导出的数据包。
4. 旧 `homepage:reset-backup:v1` 兼容备份。
5. 普通同步码空间的云端密文当前版本。

普通同步码空间没有可预览明文云端历史；如果本机没有本地历史，需要依赖同步码当前云端版本或用户已有数据包。

### 4. 恢复到本机

- 打开数据恢复中心。
- 对候选版本做完整预览。
- 确认分组、网站、组件、主题、Banner/背景和更新时间。
- 点击恢复并确认。
- 恢复后保持同步暂停，不立即覆盖云端。

### 5. 决定是否覆盖云端

只有在确认本机恢复结果正确后，才考虑上传本地覆盖云端。

上传前必须确认：

- 当前本机首页是目标恢复版本。
- 数据恢复中心已有被覆盖云端版本的本地快照。
- 当前空间不是普通同步码隐私边界被误识别的账号托管空间。
- 用户明确知道上传会影响其他设备。

### 6. 复盘

每次 P0 事故或演练失败后记录：

- 触发入口。
- 预期保护点。
- 实际保护结果。
- 是否生成本地快照。
- 是否生成云端快照。
- 是否写入本地审计或云端审计。
- 用户是否能在 3 分钟内找到恢复入口。
- 是否需要补充自动化测试、UI 文案或保护入口。

## 发布阻断条件

以下任一项失败，应阻止发布：

- 有效用户首页被覆盖前没有本地快照，且操作仍继续。
- 系统默认页、空白页或未编辑模板页自动上传覆盖云端有效首页。
- 本地覆盖云端前未保护当前云端版本。
- 数据恢复中心无法预览完整版本。
- 从本地或云端历史恢复后静默上传覆盖云端。
- 账号托管云端历史保存有效用户首页失败但上传仍成功。
- 普通同步码空间出现可预览的云端明文历史。
- 数据包导出包含同步码、账号托管 secret、登录 session 或云端历史 `document_json`。
- Supabase RLS 验证显示用户可读取他人凭证、快照或审计记录。

## Phase 1.11.7 验收标准

- 回归指南覆盖首页清空、恢复默认、模板、导入、数据包恢复、空间切换、同步冲突和多设备覆盖。
- 指南明确系统态、有效用户首页、账号托管空间和普通同步码空间的不同保护预期。
- 指南包含发布阻断条件，避免 P0 检查变成只看 happy path。
- 指南包含事故现场冻结、证据采集、恢复来源选择、恢复到本机、是否覆盖云端和复盘步骤。
- 主计划和 Phase 1.11 实施记录将 Phase 1.11.7 标记为已完成；后续主线已完成 Phase 1.11.8 基础埋点和 Phase 1.11.9 错误监控，下一步进入 Phase 1.12 组件设计和优化。

# 同步码使用指南

## Summary

同步码用于在不同设备、不同浏览器之间同步同一份首页配置。你可以在一台设备上创建同步码，再把这个同步码输入到另一台设备中。绑定后，两边可以通过手动“上传”和“拉取”交换最新的首页内容。

当前阶段已经支持基础自动同步：绑定同步码后，本地修改会在短暂延迟后自动上传；页面刷新或启动时会主动完整拉取一次云端数据；浏览器窗口重新获得焦点、页面从隐藏变为可见，或页面保持打开时，会先检查云端 revision，只有发现云端变化才完整拉取。`上传` 和 `拉取` 按钮仍然保留，作为手动兜底。

例外：如果已绑定同步空间时执行“清空内容并恢复默认”，系统会暂停自动同步，避免默认首页立刻覆盖云端。此时需要在同步面板中选择 `上传默认`、`拉取云端`、`解除本机` 或 `恢复备份`。

账号托管空间不显示完整同步码。登录账号后，可以在“首页空间”中直接恢复账号托管空间到当前浏览器；普通同步码空间仍需要输入完整同步码才能激活。

如果当前普通同步码空间已认领到账号，可以在“首页空间”中选择 `迁移为账号托管`。迁移后账号会保存托管恢复凭证，后续空白设备可登录账号恢复；旧同步码本阶段仍然有效，不会自动废弃。

登录账号后，“首页空间”列表支持重命名、设为默认和从账号移除。从账号移除普通同步码空间不会废弃同步码；从账号移除账号托管空间会删除账号侧托管恢复凭证，空白设备将不能再通过账号恢复该空间，但底层同步空间不会被删除或废弃。

## 基本概念

- **本地首页**：当前浏览器里的首页数据，保存在当前浏览器本地。
- **同步码**：用于绑定同一个云端同步空间的凭证。
- **上传**：把当前浏览器里的首页内容保存到云端。
- **拉取**：从云端读取最新首页内容，并覆盖当前浏览器里的首页。
- **解除本机**：当前浏览器不再绑定同步码，但云端数据仍保留。
- **废弃同步码**：让这个同步码彻底失效，所有设备都不能继续使用它同步。
- **从账号移除空间**：删除账号中的首页空间索引。它不同于废弃同步码，也不同于解除本机，不会删除底层同步空间。

## 在第一台设备上创建同步码

1. 打开本地或线上首页。
2. 点击右上角 `编辑`。
3. 找到 `同步码` 区域。
4. 点击 `创建`。
5. 页面会生成一串同步码。
6. 点击 `复制`，并把同步码保存到安全位置。

创建完成后，当前浏览器已经绑定这个同步码。此时当前首页内容已经上传到云端一次。

## 在第二台设备或浏览器上绑定同步码

1. 在另一台设备、另一个浏览器，或无痕窗口中打开首页。
2. 点击右上角 `编辑`。
3. 找到 `同步码` 区域。
4. 把第一台设备复制的同步码粘贴到 `输入同步码`。
5. 点击 `绑定`。
6. 浏览器会提示：绑定会用云端首页覆盖当前本地首页。
7. 确认后，当前设备会拉取云端首页并完成绑定。

绑定成功后，这台设备就和第一台设备使用同一个同步空间。

注意：同步码绑定和创建依赖浏览器 Web Crypto 加密能力。请在 `https://` 页面、`localhost` / `127.0.0.1` 本地开发地址，或支持 SubtleCrypto 的现代浏览器中使用。普通 `http://` 非安全页面或部分内嵌 WebView 可能无法完成同步码加密。

## 在一台设备修改后同步到云端

假设你在设备 A 上修改首页内容：

1. 打开设备 A 的首页。
2. 点击 `编辑`。
3. 新增、编辑、删除或排序分组和网站。
4. 确认页面显示已保存。
5. 停止操作约 2 秒，页面会自动上传。
6. 页面提示同步成功后，云端就是设备 A 的最新版本。

如果你想立即同步，也可以在 `同步码` 区域点击 `上传`。

## 在另一台设备获得最新改动

假设设备 A 已经上传了最新内容，现在要让设备 B 获取这些改动：

1. 打开设备 B 的首页。
2. 页面启动时会自动检查云端更新。
3. 如果设备 B 已经打开，可以切换到该浏览器窗口，窗口重新获得焦点时会自动检查云端更新；页面保持打开时也会每 1 分钟检查一次。
4. 如果需要立即检查，点击 `编辑`，在 `同步码` 区域点击 `拉取`。
5. 拉取成功后，设备 B 的首页会更新为云端版本。

注意：`拉取` 会用云端数据覆盖当前浏览器本地数据。如果设备 B 上有未上传的本地修改，建议先确认是否需要保留。

自动检查使用轻量 revision check；云端没有变化时不会下载完整首页文档，也不会更新 `last_pulled_at`。

## 推荐使用流程

### 单设备修改，多设备查看

1. 平时只在一台主设备上编辑首页。
2. 修改后等待自动上传完成。
3. 其他设备打开页面或重新聚焦浏览器窗口后，会自动检查更新。

这是当前阶段最稳定的使用方式。

### 多设备都可能修改

如果多台设备都会修改首页，建议遵守这个顺序：

1. 修改前先点击 `拉取`，确保当前设备是最新版本。
2. 修改首页内容。
3. 修改完成后等待自动上传，或手动点击 `上传`。
4. 换到另一台设备时，先点击 `拉取` 再继续修改。

这样可以降低冲突概率。

## 冲突情况

当前版本已经能检测基础冲突，但不会自动合并。

常见冲突场景：

1. 设备 A 和设备 B 都绑定了同一个同步码。
2. 两台设备都没有先拉取最新版本。
3. 设备 A 修改后点击 `上传`。
4. 设备 B 也修改并点击 `上传`。
5. 设备 B 可能看到冲突提示。

遇到冲突时，当前建议：

1. 如果设备 B 的修改不重要，点击 `使用云端版本`。
2. 如果设备 B 的修改更重要，点击 `本地覆盖云端`。
3. 如果不确定，先选择 `暂不处理` 或不要继续操作，手动确认需要保留的内容。

## 解除本机绑定

如果你只想让当前浏览器停止使用同步码：

1. 点击 `编辑`。
2. 找到 `同步码` 区域。
3. 点击 `解除本机`。
4. 确认操作。

解除后：

- 当前浏览器会回到本地模式。
- 当前浏览器里的首页内容仍然保留。
- 云端同步空间不会删除。
- 其他已绑定设备仍然可以继续同步。

## 管理账号首页空间

登录账号后，可以在 `编辑` -> `首页空间` 中管理账号下的首页空间。

- `设默认`：只更新账号默认空间，不拉取云端首页，也不覆盖当前浏览器本地首页。
- `重命名`：只修改账号空间名称，不影响同步码、账号托管凭证或首页内容。
- `从账号移除`：从账号空间列表中移除该空间。

移除后的影响取决于空间类型：

- 普通同步码空间：同步码本身和云端内容仍然保留；当前浏览器如果已经绑定该同步码，也可以继续同步。
- 账号托管空间：账号侧托管恢复凭证会删除；之后空白设备不能再通过账号恢复该空间。
- 当前本机正在使用的账号托管空间不能直接移除，需要先解除本机或切换到其他空间。

`从账号移除` 不会删除或废弃底层同步空间，也不会执行密钥轮换。

## 废弃同步码

如果你认为同步码已经泄露，或者不再希望任何设备继续使用它：

1. 点击 `编辑`。
2. 找到 `同步码` 区域。
3. 点击 `废弃同步码`。
4. 确认操作。

废弃后：

- 这个同步码会失效。
- 所有设备都不能继续用它上传或拉取。
- 如果还需要同步，需要重新创建新的同步码。

## 安全注意事项

- 同步码等同于该首页的读取和编辑权限。
- 不要把同步码发给不可信的人。
- 不要把同步码发到公开聊天、公开仓库或网页中。
- 普通首页网址本身不具备编辑权限。
- 如果怀疑同步码泄露，应立即点击 `废弃同步码`，然后重新创建。

## 当前限制

- 当前版本不支持自动实时同步。
- 当前版本不支持多用户权限管理。
- 当前版本不支持只读分享链接。
- 当前版本不支持自动合并多设备同时编辑产生的冲突。
- 同步 revision 当前上限为 `999`，超过后会回到 `0`；这是内部同步版本号，不影响正常使用。

这些能力会在后续登录同步和会员功能阶段继续完善。

# Phase 1.9 Implementation Plan：页面布局与导入需求集

## 阶段定位

Phase 1.9 不再只保留“浏览器收藏/标签导入”一个大块，而是拆成两个连续部分：

- Phase 1.9A：前端页面布局和 UI/UX 优化。先降低现有功能的编辑、创建和配置成本，让首页和设置页更适合日常重复使用。
- Phase 1.9B：浏览器收藏/标签导入需求集。在 UI/UX 基础收口后，再专门设计权限边界、大批量导入、预览确认和回滚策略。

Phase 1.9A 原则上不新增 Supabase 表、不新增 Storage bucket、不改变 `HomeDocumentV2` 的核心同步协议。Phase 1.9B 在需求设计阶段优先产出方案和边界，不默认承诺浏览器扩展或云端导入服务。

## Phase 1.9A：前端页面布局和 UI/UX 优化

### Phase 1.9.0：设置页信息架构收口

目标是减少设置页重复信息，让账号、空间和高级操作更像一个稳定后台，而不是多个历史阶段功能的堆叠。

任务范围：

- 将“账号托管状态”合并到“账号”栏目中，保留登录状态、账号托管可用性、Supabase 未配置状态和必要的禁用原因。
- 将“离线同步码与恢复”和“配置文件”合并到“高级操作”栏，统一承载低频、恢复类、导入导出类动作。
- 保留未登录、本地模式、同步码模式、账号托管模式下的关键状态提示，但避免同一状态在多个面板重复出现。

影响：

- 前端：主要调整设置页组件组合、文案和状态展示优先级。
- 后端：无新增 SQL，无新增 RPC。
- 架构：不改变同步码、账号托管空间或账号偏好模型。

风险：

- 不能因为合并面板而隐藏“当前是否账号托管”“是否已登录”“Supabase 是否未配置”等关键诊断信息。
- 高级操作区需要明确危险操作和普通导出操作的层级，避免误操作。

### Phase 1.9.1：首页空间创建流程弹窗化

目标是让“使用当前首页创建空间”和“从模板创建新空间”两个动作真正成为可完成的流程，而不是在设置页里直接铺开所有表单和说明。

任务范围：

- “使用当前首页创建空间”点击后弹出对话框：
  - 支持输入空间名称。
  - 支持使用默认名称。
  - 支持取消和确认。
  - 确认后复用当前首页文档创建账号托管空间。
- “从模板创建新空间”点击后弹出对话框：
  - 包含必要说明。
  - 支持模板选择。
  - 支持取消和确认。
  - 确认后使用模板生成普通 `HomeDocumentV2`，再进入现有账号托管空间创建链路。
- 设置页只保留两个入口按钮和简短状态，不直接展开完整创建细节。

影响：

- 前端：新增或复用设置页 modal/dialog 状态，复用模板库和空间创建逻辑。
- 后端：无新增 SQL，继续使用现有账号托管空间创建流程。
- 架构：不引入新的空间类型；模板结果仍是普通首页文档。

风险：

- 当前首页创建和模板创建都要保持失败回滚语义，不能创建半个空间或错误切换当前空间。
- 默认空间、当前空间、创建后是否激活等行为必须和现有首页空间管理一致。

### Phase 1.9.2：Banner/背景设置面板布局优化

目标是解决图片设置区元素挤压、按钮被挡和状态标识不齐的问题。

任务范围：

- Banner 和背景两块保持上下排布，每块独占横向空间。
- 每块左上角显示“Banner”或“背景”标签。
- 预览图占满一行。
- 上传、清除、外链 URL、遮罩强度等剩余元素置于下方，占满一行并可响应式换行。
- “未设置”等状态标识与“上传”“清除”按钮保持同一操作高度。

影响：

- 前端：主要调整 `ThemeImagePanel` 及相关 CSS。
- 后端：无新增 SQL，无新增 Storage policy。
- 架构：继续使用 Phase 1.8.1 的图片 asset 和遮罩强度字段。

风险：

- 长 URL、未登录禁用态、上传错误、窄屏和紧凑密度都要回归。
- 预览图尺寸需要稳定，避免图片加载前后导致设置页跳动。

### Phase 1.9.3：首页网站收集区编辑入口优化

目标是让新增网址和新增分组更低成本，同时保持首页默认浏览状态干净。

桌面端方案：

- 顶层“未分组”栏最前方的“增加网址”按钮改为圆形，直径与网站标签高度一致。
- 默认状态下按钮弱化或隐藏，网站列表靠左排列。
- 当鼠标 hover 到该栏最前方的触发区域，或键盘 focus 进入该区域时，网站列表向右滑动，显示圆形“增加网址”按钮。
- 页面底部“新增分组”按钮也改为同规格圆形按钮，默认弱化或隐藏，hover/focus 对应区域时显示。
- 动画要尊重 `prefers-reduced-motion`，关闭动效时直接显示/隐藏，不做滑动。

触屏端方案：

- 触屏设备没有 hover，新增网址和新增分组不能完全隐藏。
- 使用输入能力而不是只用屏幕宽度判断：
  - `(hover: hover) and (pointer: fine)`：桌面细指针设备使用 hover/focus reveal。
  - `(hover: none), (pointer: coarse)`：触屏或粗指针设备保持圆形按钮可见。
- 触屏端不做“列表先靠左、触发后再挤出按钮”的滑入逻辑；入口应作为稳定 inline 控件出现，避免用户不知道哪里可以点。
- 触屏按钮点击目标按至少 44px 级别设计，并提供明确的 `aria-label`。
- 不把长按、横向滑动露出按钮作为唯一入口；这类手势可作为增强，但不能承担主路径。

影响：

- 前端：调整首页网站列表、未分组区和新增分组入口的 DOM/CSS 结构。
- 后端：无新增 SQL。
- 架构：不改变 group/site 数据结构，只改变编辑入口的可发现性和布局。

风险：

- 网站列表滑动不能造成布局抖动、文本遮挡或移动端横向溢出。
- 需要同时覆盖鼠标、触屏、键盘和 reduced motion。
- 首页直编按钮不能遮挡网站标签点击区域。

### Phase 1.9.4：UI/UX 回归与可访问性收口

目标是给 Phase 1.9A 的跨页面布局调整做集中回归。

任务范围：

- 检查首页、设置页、模板弹窗、空间创建弹窗、图片设置面板在桌面、平板和移动端的布局。
- 检查浅色/深色、6 个主题 preset、背景图存在、紧凑密度、serif/mono 字体下的按钮和文本溢出。
- 检查键盘 Tab 顺序、焦点样式、触屏目标尺寸、hover/focus 状态和 disabled 状态。
- 检查 Supabase 未配置、未登录、上传失败、空间创建失败等错误提示是否仍然可读。

影响：

- 前端：CSS 和局部组件 polish。
- 后端：无新增。
- 架构：无新增。

风险：

- 改动范围横跨首页和设置页，必须避免局部修复引入新的重叠或按钮不可达问题。

## Phase 1.9B：浏览器收藏/标签导入需求集

### Phase 1.9.5：收藏/标签导入需求设计

目标是先明确浏览器能力边界，避免把普通网页不能做到的能力设计成主路径。

关键结论：

- 普通网页不能直接读取用户浏览器收藏夹。
- 普通网页也不能直接枚举当前浏览器已打开标签页。
- 可行入口应从用户主动提供数据开始：书签 HTML 文件、粘贴 URL 列表，或后续单独开发浏览器扩展。

需要产出：

- 导入入口对比：书签 HTML、URL 列表粘贴、浏览器扩展。
- 导入数据模型草案：原始条目、规范化 URL、标题、来源文件/来源批次、候选分组、去重状态。
- 安全边界：不上传用户原始收藏文件，除非用户明确确认；导入前本地解析和预览优先。
- MVP 选择建议：优先考虑书签 HTML 导入和 URL 列表粘贴，浏览器扩展作为后续能力。

### Phase 1.9.6：大批量导入体验设计

目标是让上千收藏也可控，避免“一键导入后首页爆炸”。

设计方向：

- 本地解析后先进入导入草稿，不直接写入正式首页。
- 提供统计摘要：总数、可导入数、重复数、无效 URL 数、候选分组数。
- 提供去重策略：同 URL、同 host、同标题近似项的不同处理级别。
- 提供分组映射：保留书签文件夹、合并到现有分组、创建新分组、导入到未分组。
- 提供抽样预览和搜索过滤；大量条目需要虚拟列表或分页。
- 提供分批确认，必要时支持撤销最近一次导入。

需要重点评估：

- 是否需要临时导入草稿状态进入 localStorage。
- 是否需要为导入批次生成 metadata，便于撤销。
- 是否需要把导入过程拆成 parser、preview、commit 三层。

### Phase 1.9.7：导入 MVP 实现候选

Phase 1.9.7 在 Phase 1.9.5 和 Phase 1.9.6 设计完成后再决定是否进入实现。

候选 MVP：

- 书签 HTML 文件导入：
  - 用户主动上传浏览器导出的 bookmarks HTML。
  - 前端本地解析 folder/title/url。
  - 进入预览和确认后写入 `HomeDocumentV2.groups[].sites`。
- URL 列表粘贴：
  - 支持一行一个 URL。
  - 自动补齐标题可先用 host 兜底，不强依赖远程抓取。
  - 适合少量迁移和跨应用复制。

暂不优先：

- 浏览器扩展：能力更完整，但涉及单独发布、权限说明、跨浏览器适配和隐私信任，适合导入需求集收口后再评估。
- 云端批处理：MVP 阶段没有必要为了解析收藏文件引入服务器任务。

## 验收标准

- Phase 1.9A 文档和计划表明确拆分为页面/UI 优化与导入需求集。
- 设置页信息架构、首页空间创建、Banner/背景面板和网站收集区编辑入口各自有明确子阶段。
- 触屏设备的新增入口不依赖 hover，不隐藏关键动作。
- Phase 1.9B 明确普通网页不能直接读取浏览器收藏夹或标签页。
- 大批量导入方案先覆盖解析、去重、预览、分组映射、确认和回滚，再决定实现入口。

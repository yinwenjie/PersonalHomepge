# Phase 1.9.6 Bulk Import Experience Design：大批量导入体验设计

## 阶段定位

Phase 1.9.6 承接 Phase 1.9.5 的收藏/标签导入需求设计，重点解决“用户有上千收藏时，如何安全、可控、低成本地导入首页”的体验和工程方案。本阶段仍然只产出设计文档，不实现导入 UI、解析器或数据写入逻辑。

Phase 1.9.5 已确认：

- 普通网页不能直接读取浏览器收藏夹、当前打开标签页或历史记录。
- MVP 推荐从用户主动提供的数据开始：书签 HTML 文件和 URL 列表粘贴。
- 导入应先进入本地草稿，确认后再写入 `HomeDocumentV2.groups[].sites`。
- 不上传原始收藏文件，不新增 Supabase 表、RPC、Storage bucket 或首页文档 schema。

Phase 1.9.6 在此基础上进一步定义：

- 上千条收藏的导入向导和信息架构。
- 草稿保存、恢复和清除策略。
- 统计摘要、筛选、搜索、预览和批量选择。
- 分组映射、去重处理和提交确认。
- 最近一次导入撤销策略。
- 性能边界、可访问性和测试样例。

## 核心结论

- 大批量导入不能做成“一键全部写入”。必须先生成导入草稿，再由用户预览和确认。
- MVP 推荐用“分页列表 + 搜索过滤 + 分组摘要”处理 1,000 到 5,000 条收藏；虚拟列表作为后续增强，不作为首版硬依赖。
- 草稿可以临时写入 localStorage，但只保存解析后的最小草稿，不保存原始 HTML 文件；草稿默认只保留当前浏览器，设置 24 小时 TTL，并提供清除入口。
- 去重默认保守：当前首页已有的相同 URL 默认不选中，同一导入批次重复 URL 只保留第一次，host 相同但路径不同只提示不默认跳过。
- 分组映射必须在提交前完成。默认用书签叶子文件夹生成候选首页分组，标题相同则合并到现有分组。
- 撤销推荐采用“最近一次导入本地撤销记录”，记录提交前快照和新增 group/site id；如果导入后又发生编辑，优先提供选择性移除本批次新增项，而不是盲目整份回滚。
- Phase 1.9.7 的 MVP 可以实现书签 HTML 导入和 URL 列表粘贴，但应保留软限制，避免一次导入过多链接导致首页不可维护。

## 体验目标

- 用户能在导入前理解规模：总数、有效数、重复数、无效数、候选分组数和将新增的网站数。
- 用户能快速定位问题：重复项、无效 URL、超大分组、未映射分组和未选中项。
- 用户能按分组处理大量收藏，而不是只能逐条编辑。
- 用户能用搜索和过滤检查关键链接是否会导入。
- 用户能控制导入范围，避免首页突然出现几千个网站。
- 用户误导入后能撤销最近一次导入。
- 用户关闭设置页或刷新页面后，短时间内可以恢复导入草稿。
- 触屏、键盘和小屏设备也能完成导入主流程。

## 非目标

- 不设计自动读取浏览器收藏夹或当前标签页。
- 不设计浏览器扩展 UI。
- 不设计 AI 自动分类。
- 不设计远程抓取网页标题、摘要、favicon 或截图。
- 不设计云端导入任务队列。
- 不将导入历史作为账号级长期记录。
- 不在 `HomeSite` 中新增 import batch 字段。
- 不承诺一次性导入 10,000 以上收藏仍然完全流畅。

## 推荐导入向导

大批量导入建议使用 5 步向导。每一步都应可取消，并且在取消前提示是否保留本地草稿。

### Step 1：选择来源

入口：

- 选择书签 HTML 文件。
- 粘贴 URL 列表。

辅助信息：

- 明确说明普通网页不能自动读取浏览器收藏夹。
- 说明书签文件会在本浏览器本地解析，不上传原始文件。
- 提供简短的浏览器导出提示入口，详细导出指南可后续补充。

校验：

- 文件类型允许 `.html`、`.htm` 和 `text/html`。
- 文件大小初始上限建议 10 MB。
- 粘贴内容初始上限建议 5,000 行。

### Step 2：解析与摘要

解析完成后进入摘要页，不直接显示所有条目。

摘要区显示：

- 总条目数。
- 有效 URL 数。
- 默认选中数。
- 当前首页重复 URL 数。
- 导入批次内重复 URL 数。
- host 相同提示数。
- 无效 URL 数。
- 候选分组数。
- 预计新增分组数。
- 预计新增网站数。

摘要页同时显示风险提示：

- 如果预计新增网站超过 500 个，提示建议分批导入。
- 如果预计新增分组超过 30 个，提示建议先做分组合并。
- 如果无效项比例很高，提示检查文件格式。
- 如果重复项比例很高，提示默认会跳过硬重复。

### Step 3：分组映射

分组映射是大批量体验的核心。用户应先按分组处理，再进入条目级预览。

每个候选分组显示：

- 来源路径。
- 默认目标分组。
- 该组总数、默认选中数、重复数、无效数。
- 前 3 到 5 个样例链接。
- 映射模式。

映射模式：

- 创建新分组。
- 合并到现有分组。
- 导入到未分组。
- 跳过该来源分组。

默认策略：

- 来源文件夹叶子名与现有分组标题完全相同时，默认合并。
- 来源文件夹为空时，默认导入到未分组。
- 来源分组没有有效条目时，默认跳过。
- 其他来源分组默认创建新分组。

批量操作：

- 所有同名分组合并到现有分组。
- 所有新分组改为导入到未分组。
- 所有空分组跳过。
- 只导入选中的若干分组。
- 分组标题批量去前缀，例如去掉 `Bookmarks Bar /`。

### Step 4：条目预览

条目预览用于处理细节，不应要求用户逐条看完。

推荐布局：

- 顶部固定摘要条：选中数、将新增数、重复数、无效数。
- 左侧或顶部筛选：状态、来源分组、目标分组。
- 主列表：分页展示条目。
- 右侧或底部批量操作：选择当前筛选结果、取消选择、只选新链接、跳过当前分组。

列表字段：

- 勾选状态。
- 标题或兜底名称。
- URL 和 hostname。
- 来源路径。
- 目标分组。
- 状态标签：新链接、当前首页重复、批次内重复、host 相同、无效。
- 简短原因。

筛选项：

- 全部。
- 已选。
- 未选。
- 新链接。
- 当前首页重复。
- 批次内重复。
- host 相同。
- 无效 URL。
- 未映射分组。

搜索：

- 搜索标题、URL、hostname、来源路径和目标分组。
- 输入 debounce。
- 搜索结果支持“全选当前结果”和“取消选择当前结果”。

分页策略：

- MVP 推荐每页 100 条。
- 用户可切换 50、100、200。
- 当前筛选结果超过 2,000 条时，保留分页，避免一次渲染全部 DOM。
- 虚拟列表可作为后续增强，但首版不要求引入新依赖。

### Step 5：确认导入

确认页必须提供最终 diff 摘要：

- 将新增网站数。
- 将新增分组数。
- 将合并到现有分组的网站数。
- 将跳过的重复项数。
- 将跳过的无效项数。
- 当前首页网站总数。
- 导入后首页网站总数。

确认按钮文案应明确，例如“确认导入 286 个网站”，不要只写“确认”。

确认前校验：

- 选中数为 0 时禁止提交。
- 选中数超过建议上限时显示强提示。
- 如果存在未映射的已选条目，禁止提交或要求先选择目标。
- 如果当前首页在导入草稿生成后发生变化，提示用户重新检查重复项和目标分组。

## 草稿保存与恢复

### 是否进入 localStorage

建议进入 localStorage，但仅保存最小导入草稿：

- 保存解析后的条目、分组映射、选择状态和统计摘要。
- 不保存原始书签 HTML 文件。
- 不保存 favicon data URL。
- 不保存浏览器扩展原始 payload。
- 不同步到云端。
- 不进入数据导出包，除非未来明确支持本地诊断导出。

推荐 key：

```text
homepage:bookmark-import-draft:v1
homepage:bookmark-import-undo:v1
```

草稿生命周期：

- 只保留最近一个草稿。
- TTL 24 小时。
- 导入成功后默认清除草稿。
- 用户取消时询问“保留草稿”或“丢弃草稿”。
- 设置页高级操作中可提供“清除导入草稿”。

恢复策略：

- 打开导入向导时，如果发现未过期草稿，显示“继续上次导入”。
- 如果当前首页 documentId 与草稿记录不一致，提示这是另一个首页空间的草稿，默认不继续。
- 如果当前首页 revision 与草稿生成时不同，允许继续，但必须重新计算重复状态和目标分组。
- 如果 localStorage 容量不足，降级为内存草稿，并提示刷新页面会丢失。

草稿记录建议字段：

```ts
export interface BookmarkImportDraftStorageV1 {
  schema: "homepage-bookmark-import-draft-v1";
  draft: BookmarkImportDraft;
  homeDocumentId: string;
  homeRevision: number;
  savedAt: string;
  expiresAt: string;
}
```

## 去重体验

去重要可解释，不能让用户觉得系统静默丢数据。

### 状态定义

- 新链接：默认选中。
- 当前首页重复 URL：默认不选中。
- 导入批次重复 URL：第一条默认选中，后续不选中。
- host 相同：默认选中，但显示提示。
- 无效 URL：不可选中。

### 批量策略

用户可以选择：

- 只导入新链接。
- 包含 host 相同链接。
- 包含当前首页重复 URL。
- 删除当前筛选结果中的选择。
- 保留每个 URL 的第一条。

MVP 不建议做标题近似去重的自动跳过。标题近似容易误伤，只作为后续分析提示。

### 解释文案

每个重复状态都应给出短理由：

- “当前首页已存在相同 URL。”
- “本次导入中已有相同 URL，默认保留第一条。”
- “当前首页已有同一网站域名，但路径不同。”
- “仅支持 http:// 或 https:// 链接。”

## 分组映射体验

### 默认映射

默认用叶子文件夹作为候选首页分组：

```text
Bookmarks Bar / Work / Docs / API -> API
```

理由：

- 当前首页只有一级分组。
- 叶子文件夹通常更接近用户实际分类。
- 完整路径仍显示在来源路径中，避免丢上下文。

### 冲突处理

候选分组标题冲突时：

- 与现有首页分组同名：默认合并。
- 多个来源路径叶子名相同：默认合并为同一个候选分组，但在样例和来源路径中显示多个来源。
- 标题为空：默认导入到未分组。
- 标题过长：预览中显示完整标题，提交时建议用户改短。

### 大量分组

如果候选分组超过 30 个：

- 显示“分组较多”提示。
- 默认折叠分组详情，只展示摘要和样例。
- 提供按来源路径搜索分组。
- 提供批量合并到现有分组或未分组。

如果候选分组超过 100 个：

- 建议用户先在浏览器或书签管理器中整理后再导入。
- 仍允许继续，但需要明确确认。

## 批量选择与分批确认

### 批量选择

批量选择必须作用于“当前筛选结果”，并清楚说明影响范围。

推荐操作：

- 选择当前筛选结果。
- 取消选择当前筛选结果。
- 只保留新链接。
- 跳过当前目标分组。
- 跳过所有重复 URL。
- 恢复默认选择。

每次批量操作后显示结果，例如：

```text
已取消选择 184 个重复链接。
```

### 分批确认

MVP 不必实现真正的多阶段提交队列，但需要支持“分批选择并提交”的体验：

- 用户可以通过分组筛选，只选择若干分组后提交。
- 未提交的草稿可保留，方便后续继续导入剩余分组。
- 导入成功后，草稿中已提交条目标记为已导入，用户可选择继续处理剩余条目。

若 Phase 1.9.7 为了降低实现复杂度，也可以先选择“导入成功后清除草稿”，但 UI 需要鼓励用户在提交前筛好范围。更完整的分批体验可以作为 Phase 1.9.7b 或 Phase 1.10 候选。

## 提交与撤销

### 提交计划

提交前将草稿转换为 commit plan：

- 创建哪些新分组。
- 合并到哪些现有分组。
- 每个分组新增哪些网站。
- 跳过哪些重复或无效条目。

commit plan 生成后再修改 `HomeDocumentV2`，并复用现有提交链路。

### 撤销策略

推荐保存本地撤销记录：

```ts
export interface BookmarkImportUndoStorageV1 {
  schema: "homepage-bookmark-import-undo-v1";
  importBatchId: string;
  homeDocumentId: string;
  beforeRevision: number;
  afterRevision: number;
  beforeDocument: HomeDocumentV2;
  addedGroupIds: string[];
  addedSiteIdsByGroupId: Record<string, string[]>;
  createdAt: string;
  expiresAt: string;
}
```

撤销规则：

- 只保留最近一次导入的撤销记录。
- TTL 建议 24 小时。
- 如果当前文档 revision 等于导入后的 revision，允许完整恢复 `beforeDocument`。
- 如果导入后又有其他编辑，优先提供“移除本次导入新增项”，并提示这不会恢复被用户后续移动或编辑过的内容。
- 如果新增项已经被删除或移动，撤销应尽力处理，不应报错中断整个撤销。
- 撤销操作本身也走现有首页文档提交链路。

不推荐在 MVP 中把撤销记录写入云端。撤销是当前浏览器的安全网，不是长期审计历史。

## 大批量性能策略

### 初始规模边界

建议首版采用软硬限制：

| 指标 | 建议值 | 行为 |
|---|---:|---|
| 文件大小 | 10 MB | 超过后拒绝并提示分批导出或清理。 |
| 解析条目数 | 5,000 | 超过后进入受限模式或提示暂不支持。 |
| 默认建议提交数 | 500 | 超过后提示建议分批。 |
| 单次确认软上限 | 2,000 | 超过后强提示，但可继续。 |
| 每页预览数量 | 100 | 支持 50/100/200 切换。 |

这些数值不是产品承诺，Phase 1.9.7 实现时需要用真实样例验证。

### 处理策略

- 解析、URL 规范化和去重使用纯函数，方便单元测试。
- 使用 `Map` 和 `Set` 做 URL、host 和分组索引，避免 O(n²)。
- 搜索输入 debounce。
- 统计摘要基于索引增量更新，避免每次筛选重算全部字段。
- 列表渲染使用分页，不一次性渲染上千行。
- 批量选择只更新必要字段，避免每次操作复制过大的对象树。
- 解析过程中显示处理中状态。
- 如果主线程解析大文件卡顿明显，后续再评估 Web Worker。

### 虚拟列表取舍

首版不建议强依赖虚拟列表：

- 当前项目没有表格或虚拟列表基础组件。
- 分页更容易处理键盘、屏幕阅读器和移动端。
- 大批量导入是低频操作，不必为了首版引入复杂滚动状态。

后续若用户导入规模长期超过 5,000 条，可以再引入虚拟列表或 Web Worker。

## 错误与异常状态

需要覆盖：

- 文件无法读取。
- 文件过大。
- HTML 解析失败。
- 没有找到任何 URL。
- URL 全部无效。
- URL 全部重复。
- localStorage 写入失败。
- 草稿过期。
- 草稿对应的首页空间已切换。
- 当前首页在草稿生成后发生变化。
- 提交时遇到同步冲突。
- 提交成功但云同步失败。

原则：

- 解析失败不影响当前首页。
- 草稿失败不阻塞当前会话内导入。
- 提交失败不清除草稿。
- 提交成功但同步失败时，按现有同步状态提示处理。

## 移动端与可访问性

移动端策略：

- 不依赖 hover。
- 主要操作按钮固定在当前步骤底部，但不能遮挡列表内容。
- 分组映射和条目预览优先使用单列布局。
- 列表项以卡片式信息块展示，避免横向表格溢出。
- 批量操作放入菜单或底部操作区。

键盘策略：

- Stepper、筛选、搜索、分页、复选框和确认按钮都可键盘操作。
- 批量操作完成后焦点不丢失。
- 错误提示使用可感知的状态区域。
- 确认导入弹窗应有明确标题和焦点管理。

屏幕阅读器：

- 摘要数字要有文本说明。
- 状态标签不能只靠颜色。
- 复选框 label 应包含标题、URL 和状态。
- 批量操作要说明影响范围。

## 前端影响

Phase 1.9.6 不改前端代码。后续实现时建议新增：

- `domain/bookmark-import.ts`：草稿、统计、去重和 commit plan。
- `domain/bookmark-import-storage.ts`：草稿和撤销记录 localStorage。
- `domain/bookmark-import-selection.ts`：筛选、搜索和批量选择。
- `components/bookmark-import-dialog.tsx`：导入向导框架。
- `components/bookmark-import-summary.tsx`：摘要和风险提示。
- `components/bookmark-import-group-mapping.tsx`：分组映射。
- `components/bookmark-import-preview.tsx`：分页预览和筛选。
- `components/bookmark-import-confirm.tsx`：最终 diff 和确认。

实现时应复用：

- `HomeDocumentV2`、`HomeGroup`、`HomeSite`。
- `createId`。
- `renumberGroups`。
- `renumberSites`。
- 当前首页文档提交和同步链路。

## 后端与 Supabase 影响

Phase 1.9.6 推荐方案不新增后端能力：

- 不新增 Supabase 表。
- 不新增 RPC。
- 不新增 Storage bucket。
- 不新增 Edge Function。
- 不改变账号托管空间、同步码空间或加密同步协议。

导入确认后只是当前首页文档的一次普通编辑：

- 本地模式写入 localStorage。
- 同步码模式继续端到端加密后上传。
- 账号托管模式继续走当前账号托管首页空间。

需要注意首页文档体积变大后的同步成本。Phase 1.9.7 实现时应在确认页提示导入数量过大可能增加同步时间。

## 测试样例

Phase 1.9.7 实现前应准备样例：

- Chrome/Edge 书签 HTML：100 条、1,000 条、5,000 条。
- Firefox 书签 HTML：含多级文件夹和空文件夹。
- Safari 书签 HTML：含收藏栏和阅读列表相似结构。
- 含大量 favicon data URL 的 HTML。
- 深层文件夹超过 5 级。
- 重复 URL 分布在不同文件夹。
- 同 host 不同 path。
- 非 http/https URL：`javascript:`, `mailto:`, `file:`, `chrome:`, `about:`。
- 标题为空。
- URL 无 scheme。
- URL 列表粘贴：纯 URL、空行、重复 URL、Markdown 链接、混合文本。
- 当前首页已有 0、100、500 个网站的重复检测。
- localStorage 容量不足。
- 草稿过期。
- 导入后撤销。
- 导入后继续编辑再撤销。

## Phase 1.9.7 MVP 建议

Phase 1.9.7 可以进入实现的最低可用版本：

- 入口放在设置页高级操作中。
- 支持书签 HTML 文件导入。
- 支持 URL 列表粘贴。
- 前端本地解析。
- 生成导入草稿。
- 摘要页展示统计和风险提示。
- 分组映射支持创建、合并、未分组和跳过。
- 条目预览使用分页，不引入虚拟列表。
- 支持搜索、状态筛选和基础批量选择。
- 确认后写入当前首页。
- 保存最近一次导入撤销记录。
- 草稿 localStorage 24 小时 TTL。

首版可以暂缓：

- 真正的多批次提交队列。
- 虚拟列表。
- Web Worker。
- 标题近似去重。
- AI 分类。
- 浏览器扩展。
- 云端导入历史。

- 文档定义分页优先、虚拟列表后置的性能方案。
- 文档定义最近一次导入撤销策略。

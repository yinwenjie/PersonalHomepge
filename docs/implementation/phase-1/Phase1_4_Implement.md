# Phase 1.4 实施计划：前端展示页与设置页优化

## Summary

Phase 1.4 只做前端展示页和 UI 交互优化。Supabase Auth + 账号首页同步整体顺延到 Phase 1.5。

本阶段将首页拆成两个更清晰的前端体验：

- 日常首页：保持干净、快速、适合每天打开。
- 设置页：通过右侧状态区的齿轮图标进入，用于管理账号占位、同步码、配置文件和通用设置占位。

本阶段不新增数据库、不实现 Supabase Auth、不改变同步码协议、不改变 `HomeDocumentV2` schema。

## Goals

- 降低用户维护网站收藏的操作成本。
- 让日常首页减少管理控件干扰。
- 将同步码、配置导入导出和恢复默认整理到独立设置页。
- 为 Phase 1.5 的账号登录和账号同步预留清晰 UI 入口。
- 保留当前搜索、拖拽、同步码、导入导出和恢复默认能力。

## Key Decisions

- 日常首页不再显示右上角状态 pill 和文字“编辑”按钮。
- 右侧状态展示区继续作为固定区域，用户名占位符仍为 `Local`。
- 编辑入口改为典型齿轮图标，放在右侧状态展示区中 `Local` 的右侧。
- 同步状态并入右侧状态展示区已有的 `synced` 状态行，不再在顶部重复展示。
- 点击齿轮进入设置页，而不是在日常首页展开完整编辑模式。
- 设置页中设计登录界面，但只做静态 UI，不接 Supabase Auth。
- “未分组”作为网站收集区的固定首区，用于保存不属于任意自定义分组的网站。
- 日常首页通过分组和网站的收起态浮层菜单完成新增、编辑、删除和拖动。
- 设置页不再承担具体网站和分组编辑功能，只作为后台设置入口。

## Planned Changes

- 首页网站收集区：
  - 增加固定的“未分组”区域，始终位于所有分组最上方。
  - “未分组”横向布满整个网站收集区，不使用左侧分组标题 + 右侧网格的旧布局限制。
  - “未分组”保存所有不属于任意用户分组的网站。
  - 日常首页只展示未分组网站磁贴，不显示“未分组”标题、管理按钮或外层卡片。
  - “未分组”左侧保留一个窄 `+` 按钮，用于新增未分组网站。
  - 分组末尾不显示“新增网站”按钮。
  - 每个分组和网站的新增、修改、删除操作在首页浮层菜单中完成。

- 顶部与右侧状态区：
  - 移除顶部右侧同步状态 pill。
  - 移除顶部右侧文字“编辑/完成”按钮。
  - 将同步状态合并到右侧状态展示区的状态行。
  - 在 `Local` 用户名占位符右侧增加齿轮编辑图标。
  - 齿轮图标使用按钮语义和可访问 label。

- 设置页：
  - 通过齿轮图标进入独立设置页。
  - 设置页只包含后台管理模块：
    - 账号登录占位区。
    - 同步码。
    - 导入/导出 JSON 和恢复默认。
    - 通用设置占位区。
  - 设置页不展示搜索框、网站收集区、分组列表、右侧状态区或组件列表。
  - 设置页包含同步码管理能力：
    - 创建同步码。
    - 输入同步码绑定。
    - 上传、拉取、解除本机、废弃同步码。
    - 冲突处理继续沿用现有逻辑。
  - 设置页设计登录区：
    - 展示邮箱输入、登录按钮、账号同步说明或占位状态。
    - 所有登录控件为不可用或明确标记为“即将支持”。
    - 不调用 Supabase Auth，不保存账号状态。

- 数据与保存：
  - 继续使用现有本地保存入口。
  - 所有编辑保存后仍触发现有同步码自动上传。
  - 不修改 `HomeDocumentV2` schema。
  - “未分组”优先用前端约定实现；如需要持久化，也必须复用现有 `groups/sites` 结构，不新增 schema 字段。

## Subtask Breakdown

### Phase 1.4.1：首页信息架构调整

- 移除顶部同步状态 pill 和文字编辑按钮。
- 将同步状态展示合并进右侧状态展示区。
- 在 `Local` 右侧增加齿轮编辑入口。
- 确保搜索区、网站收集区和右侧状态/组件区域位置不大幅改变。
- 实现时保留现有同步码后台 effect，避免因为隐藏同步码面板导致自动同步停止。
- 验收口径：
  - 首页顶部只保留日期和 `Home` 标题。
  - 右侧状态区能看到当前 sync status。
  - 齿轮按钮能进入设置页。

### Phase 1.4.2：“未分组”区域

- 在网站收集区最上方增加“未分组”。
- “未分组”横向布满网站收集区。
- 未分组区域支持展示网站磁贴。
- 日常首页不显示“未分组”标题、管理按钮或外层卡片。
- 日常首页不在未分组末尾提供新增网站按钮。
- 未分组新增入口放在该区域左侧的窄 `+` 按钮中。
- 实现方式：
  - 使用保留分组 `group-ungrouped`，不新增 schema 字段。
  - `group-ungrouped` 固定 order 为 `0`。
  - 普通分组从 `1` 开始排序。
  - “未分组”不参与普通分组拖拽排序，也不允许作为普通分组删除。
- 验收口径：
  - 旧本地数据加载后也能看到“未分组”区域。
  - 新增默认文档包含“未分组”。
  - 网站仍可拖入“未分组”。

### Phase 1.4.3：设置页路由与布局

- 新增独立设置页或等价的全页设置视图。
- 齿轮图标进入设置页。
- 设置页提供返回首页入口。
- 设置页采用极简模块化布局。
- 设置页包含四块区域：
  - 账号登录占位区。
  - 同步码管理。
  - 导入/导出 JSON 和恢复默认。
  - 通用设置占位区。
- 设置页不承载网站与分组的具体编辑能力。
- 网站与分组的首页直编体验进入 Phase 1.4.4。
- 同步码面板的设置页收纳进入 Phase 1.4.5。
- 账号区和通用设置区只展示不可用占位，不触发 Supabase Auth 或真实设置保存。
- 验收口径：
  - `/edit` 可直接打开。
  - `/edit` 能读取当前本地首页并展示账号占位、同步码、配置文件和通用设置四个模块。
  - `/edit` 能返回 `/`。
  - `/edit` 不展示搜索框、网站收集区、右侧状态区和组件列表。
  - 登录区域和通用设置区域按钮不可用或明确“即将支持”。

### Phase 1.4.4：分组与网站编辑能力

- 日常首页中每个网站磁贴提供收起态操作按钮，默认只显示 `...`，避免管理控件长期遮挡视线。
- 点击网站磁贴操作按钮后，浮层菜单向左覆盖在磁贴内部，菜单顺序为：编辑、拖动、删除。
- 网站删除入口保留在浮层菜单和“编辑网站”表单底部，两处都沿用二次确认。
- 普通分组标题区提供收起态操作按钮，点击后弹出浮层菜单。
- 普通分组菜单顺序为：编辑、拖动、`+` 新增网站、删除。
- 最顶层“未分组”不显示标题和分组操作菜单，只在左侧显示一个 `+` 新增网站按钮。
- 新增网站继续使用现有新增网站表单，不在每个分组末尾放置大号新增按钮。
- 删除操作继续二次确认。
- URL 校验沿用现有 `http://` / `https://` 规则。

### Phase 1.4.5：同步码能力迁移到设置页

- 将现有同步码面板移动或复用到设置页。
- 创建、绑定、上传、拉取、解除本机、废弃同步码行为保持不变。
- 冲突处理 UI 保持可用。
- 首页日常视图不直接展示完整同步码面板。

### Phase 1.4.6：登录占位 UI

- 在设置页设计登录区域。
- 展示账号登录和账号同步的未来入口。
- 登录按钮不实现真实功能。
- 文案明确当前阶段账号登录尚未启用，避免用户误以为可用。

### Phase 1.4.7：回归与响应式

- 验证首页日常打开效率不下降。
- 验证设置页在桌面和移动端可用。
- 验证拖拽、搜索、同步码和导入导出不回退。
- 确认所有新增按钮有明确可访问名称。

### Phase 1.4.8：首页与设置页解耦收口

- 目标：
  - 拆除 `HomeDashboard surface="edit"` 的双分支结构。
  - 让首页和设置页成为两个独立组件，但保持用户行为不变。
  - 为 Phase 1.5 账号同步降低组件耦合和回归风险。
- 代码结构：
  - 新增 `useHomeDocumentController`，集中负责本地首页加载、保存、替换、同步元数据更新、JSON 导入导出和恢复默认。
  - 新增 `useHomeDocumentEditor`，集中负责分组/网站 CRUD、编辑表单状态、URL 校验和删除确认。
  - 新增 `HomeDocumentEditorModal`，把分组/网站编辑弹窗从首页主体组件中抽出。
  - 新增 `SettingsDashboard`，独立承载账号占位、同步码、配置文件和通用设置四个模块。
  - `/edit` 继续保留原路由，但改为渲染独立设置页，不再通过 `HomeDashboard surface="edit"` 进入设置模式。
- 首页边界：
  - `HomeDashboard` 只负责首页标题、欢迎条、搜索、网站收集区、右侧状态/组件区和隐藏挂载的自动同步面板。
  - 首页继续保留网站/分组浮层菜单与编辑弹窗，因为网站收集区直编是 Phase 1.4 的核心交互。
  - 首页隐藏的 `SyncPanel visible={false}` 保留，确保自动同步 effect 不回退。
- 设置页边界：
  - 设置页不展示搜索框、网站收集区、首页右侧状态区或组件列表。
  - 设置页继续完整承载同步码功能。
  - 设置页继续提供 JSON 导入、导出和恢复默认。
  - 账号和通用设置仍为不可用占位，不接 Supabase Auth。
- 不变项：
  - 不修改 `HomeDocumentV2` schema。
  - 不修改 Supabase 表、RPC 或同步码协议。
  - 不引入账号登录逻辑。
  - 保持 revision 增长、localStorage key、v1/v2 迁移逻辑不变。
- 验收口径：
  - `/` 和 `/edit` 均可独立打开。
  - 首页搜索、DuckDuckGo 回车搜索、欢迎条、网站/分组浮层菜单、添加/编辑/删除、拖拽排序正常。
  - 设置页同步码、导入导出、恢复默认正常。
  - 自动同步仍可在首页后台运行。
  - `npm run lint`、`npm run typecheck`、`npm run build` 通过。

### Phase 1.4.9：配置文件恢复默认 UX 补漏

- 目标：
  - 降低“恢复默认”误操作造成首页内容不可找回的风险。
  - 保持实现轻量，只保存最近一次重置前备份，不做完整历史版本系统。
- 实施记录：
  - 设置页“恢复默认”按钮改名为“清空内容并恢复默认”，提升危险操作识别度。
  - 新增本地备份 key：`homepage:reset-backup:v1`。
  - 点击“清空内容并恢复默认”前，先把当前 `HomeDocumentV2` 写入本地备份。
  - 备份写入失败时取消恢复默认，不清空当前首页。
  - 设置页检测到备份后显示“恢复上一次重置前页面”按钮。
  - 当前首页内容已经等同默认内容时，禁用“清空内容并恢复默认”按钮。
  - `resetDefault()` 内部也检查默认内容，防止双击或未来其他入口把备份覆盖成默认模板。
  - 默认内容判断只比较 `groups`、`widgets` 和 `theme`，不比较 `revision`、`updatedAt`、`documentId`、`syncMeta` 或 `billing`。
- 边界：
  - 只保留最近一次重置前备份。
  - 不做历史版本列表、云端备份或账号级恢复。
  - 不改变同步码协议；恢复备份后会作为一次本地文档变更参与现有同步逻辑。
- 验收口径：
  - 非默认首页点击“清空内容并恢复默认”后，本地首页恢复默认，并出现恢复备份按钮。
  - 点击“恢复上一次重置前页面”后，能恢复清空前首页。
  - 连续点击或默认首页状态下，不会把已有备份覆盖成默认模板。
  - `npm run lint`、`npm run typecheck`、`npm run build` 通过。

## Out Of Scope

- 不改 RLS。

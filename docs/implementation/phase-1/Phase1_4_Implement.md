# Phase 1.4 实施计划：前端展示页与编辑页面优化

## Summary

Phase 1.4 只做前端展示页和 UI 交互优化。Supabase Auth + 账号首页同步整体顺延到 Phase 1.5。

本阶段将首页拆成两个更清晰的前端体验：

- 日常首页：保持干净、快速、适合每天打开。
- 编辑页面：通过右侧状态区的齿轮图标进入，用于管理分组、网站、同步码，并预留登录界面。

本阶段不新增数据库、不实现 Supabase Auth、不改变同步码协议、不改变 `HomeDocumentV2` schema。

## Goals

- 降低用户维护网站收藏的操作成本。
- 让日常首页减少管理控件干扰。
- 将同步码能力从首页编辑工具栏中整理到独立编辑页面。
- 为 Phase 1.5 的账号登录和账号同步预留清晰 UI 入口。
- 保留当前搜索、拖拽、同步码、导入导出和恢复默认能力。

## Key Decisions

- 日常首页不再显示右上角状态 pill 和文字“编辑”按钮。
- 右侧状态展示区继续作为固定区域，用户名占位符仍为 `Local`。
- 编辑入口改为典型齿轮图标，放在右侧状态展示区中 `Local` 的右侧。
- 同步状态并入右侧状态展示区已有的 `synced` 状态行，不再在顶部重复展示。
- 点击齿轮进入新的编辑页面，而不是在日常首页展开完整编辑模式。
- 编辑页面中设计登录界面，但只做静态 UI，不接 Supabase Auth。
- “未分组”作为网站收集区的固定首区，用于保存不属于任意自定义分组的网站。
- 日常首页不再在分组末尾显示“新增网站”按钮；新增、编辑、删除集中放到编辑页面，避免首页管理控件过多。

## Planned Changes

- 首页网站收集区：
  - 增加固定的“未分组”区域，始终位于所有分组最上方。
  - “未分组”横向布满整个网站收集区，不使用左侧分组标题 + 右侧网格的旧布局限制。
  - “未分组”保存所有不属于任意用户分组的网站。
  - 日常首页只展示未分组网站磁贴，不显示“未分组”标题、管理按钮或外层卡片。
  - 分组末尾不显示“新增网站”按钮。
  - 每个分组和网站的新增、修改、删除操作进入编辑页面完成。

- 顶部与右侧状态区：
  - 移除顶部右侧同步状态 pill。
  - 移除顶部右侧文字“编辑/完成”按钮。
  - 将同步状态合并到右侧状态展示区的状态行。
  - 在 `Local` 用户名占位符右侧增加齿轮编辑图标。
  - 齿轮图标使用按钮语义和可访问 label。

- 编辑页面：
  - 通过齿轮图标进入独立编辑页面。
  - 编辑页面包含网站分组管理能力：
    - 新增、修改、删除分组。
    - 新增、修改、删除网站。
    - 管理“未分组”网站。
    - 保留拖拽排序和跨组移动。
  - 编辑页面包含同步码管理能力：
    - 创建同步码。
    - 输入同步码绑定。
    - 上传、拉取、解除本机、废弃同步码。
    - 冲突处理继续沿用现有逻辑。
  - 编辑页面设计登录区：
    - 展示邮箱输入、登录按钮、账号同步说明或占位状态。
    - 所有登录控件为不可用或明确标记为“即将支持”。
    - 不调用 Supabase Auth，不保存账号状态。

- 数据与保存：
  - 继续使用现有本地保存入口。
  - 所有编辑保存后仍触发现有同步码自动上传。
  - 不修改 `HomeDocumentV2` schema。
  - “未分组”优先用前端约定实现；如需要持久化，也必须复用现有 `groups/sites` 结构，不新增 schema 字段。

## Subtask Breakdown

### Phase 1.4.1：首页信息架构调整

- 移除顶部同步状态 pill 和文字编辑按钮。
- 将同步状态展示合并进右侧状态展示区。
- 在 `Local` 右侧增加齿轮编辑入口。
- 确保搜索区、网站收集区和右侧状态/组件区域位置不大幅改变。
- 实现时保留现有同步码后台 effect，避免因为隐藏同步码面板导致自动同步停止。
- 验收口径：
  - 首页顶部只保留日期和 `Home` 标题。
  - 右侧状态区能看到当前 sync status。
  - 齿轮按钮能进入编辑页面。

### Phase 1.4.2：“未分组”区域

- 在网站收集区最上方增加“未分组”。
- “未分组”横向布满网站收集区。
- 未分组区域支持展示网站磁贴。
- 日常首页不显示“未分组”标题、管理按钮或外层卡片。
- 日常首页不在未分组末尾提供新增网站按钮。
- 没有未分组网站时不制造额外视觉负担；新增网站入口放在编辑页面。
- 实现方式：
  - 使用保留分组 `group-ungrouped`，不新增 schema 字段。
  - `group-ungrouped` 固定 order 为 `0`。
  - 普通分组从 `1` 开始排序。
  - “未分组”不参与普通分组拖拽排序，也不允许作为普通分组删除。
- 验收口径：
  - 旧本地数据加载后也能看到“未分组”区域。
  - 新增默认文档包含“未分组”。
  - 网站仍可拖入“未分组”。

### Phase 1.4.3：编辑页面路由与布局

- 新增独立编辑页面或等价的全页编辑视图。
- 齿轮图标进入编辑页面。
- 编辑页面提供返回首页入口。
- 编辑页面包含三块区域：
  - 网站与分组管理。
  - 同步码管理。
  - 登录占位区。
- 编辑页面直接复用原“编辑”模式下的分组、网站、导入导出、恢复默认和同步码能力。
- 网站与分组的进一步体验优化进入 Phase 1.4.4。
- 同步码面板的视觉重排进入 Phase 1.4.5。
- 登录区只展示不可用占位，不触发 Supabase Auth。
- 验收口径：
  - `/edit` 可直接打开。
  - `/edit` 能读取当前本地首页并展示概览、编辑工具栏、同步码面板和网站收集区。
  - `/edit` 能返回 `/`。
  - 登录区域按钮不可用，且文案明确“即将支持”。

### Phase 1.4.4：分组与网站编辑能力

- 编辑页面中每个分组提供编辑按钮，可修改分组内容或删除分组。
- 编辑页面中每个网站提供编辑按钮，可修改网站内容或删除网站。
- 新增网站继续使用编辑页面中的原有新增网站流程，不在日常首页分组末尾放置新增按钮。
- 删除操作继续二次确认。
- URL 校验沿用现有 `http://` / `https://` 规则。

### Phase 1.4.5：同步码能力迁移到编辑页面

- 将现有同步码面板移动或复用到编辑页面。
- 创建、绑定、上传、拉取、解除本机、废弃同步码行为保持不变。
- 冲突处理 UI 保持可用。
- 首页日常视图不直接展示完整同步码面板。

### Phase 1.4.6：登录占位 UI

- 在编辑页面设计登录区域。
- 展示账号登录和账号同步的未来入口。
- 登录按钮不实现真实功能。
- 文案明确当前阶段账号登录尚未启用，避免用户误以为可用。

### Phase 1.4.7：回归与响应式

- 验证首页日常打开效率不下降。
- 验证编辑页面在桌面和移动端可用。
- 验证拖拽、搜索、同步码和导入导出不回退。
- 确认所有新增按钮有明确可访问名称。

## Out Of Scope

- 不做 Supabase Auth。
- 不新增账号首页表。
- 不改 RLS。
- 不实现账号同步。
- 不实现真实登录。
- 不实现真实 Todo、日历或其他增强组件。
- 不改变 `HomeDocumentV2` schema。

## Test Plan

- 首页不再出现顶部状态 pill 和文字“编辑”按钮。
- 右侧状态展示区显示同步状态，并在 `Local` 右侧显示齿轮图标。
- 点击齿轮能进入编辑页面。
- 编辑页面能返回首页。
- “未分组”显示在所有分组上方，并横向布满网站收集区。
- 日常首页不显示“未分组”标题、管理按钮、外层卡片或分组末尾新增网站按钮。
- 编辑页面中的分组编辑按钮可进入修改或删除流程。
- 编辑页面中的网站编辑按钮可进入修改或删除流程。
- 编辑页面中的新增网站流程可继续添加网站。
- 输入非法 URL 时阻止保存并提示。
- 输入 `<script>alert(1)</script>` 只作为文本显示。
- 删除分组和网站仍二次确认。
- 分组拖拽排序、网站组内排序和跨组移动仍正常。
- 搜索过滤仍正常；搜索状态下拖拽仍禁用。
- 同步码创建、绑定、上传、拉取和自动同步不回退。
- 编辑页面登录区只展示占位 UI，不触发真实登录请求。
- 桌面、窄屏和手机宽度下控件不重叠。
- `npm run lint`、`npm run typecheck`、`npm run build` 通过。

## Assumptions

- Phase 1.4 是纯前端体验阶段。
- Phase 1.5 才开始 Supabase Auth + 账号首页同步。
- 当前同步码机制继续作为唯一云同步能力。
- “未分组”是固定首区，不参与普通分组排序。
- 编辑入口从顶部移动到右侧状态区齿轮按钮。

# 账号托管同步 Backlog

## Summary

账号同步与同步码同步需要长期并行，但登录用户的默认体验应从“手动管理同步码”升级为“账号管理首页空间”。同步码保留匿名同步、恢复和高级管理价值；账号托管空间则为已注册用户提供空白设备登录后直接恢复首页的能力。

核心决策：

- 登录用户默认创建 `account-managed` 首页空间。
- 未登录用户继续使用 `sync-code` 首页空间。
- 账号可以托管首页空间的加密凭证，凭证通过 RLS 限制为本人可读。
- 账号托管空间允许空白设备登录后直接拉取、解密和展示首页内容。
- Phase 1.11 已将账号托管空间的目标安全模型调整为“账号可信托管 + 可恢复/可审计记录”；Phase 1.11.5 已先落地账号托管云端历史版本 v1，Phase 1.11.6 已收口账号托管、普通同步码和高隐私模式边界，普通同步码空间继续保持密文边界。
- 同步码空间登录后支持两种处理：仅认领索引，或迁移为账号托管空间。
- 高隐私空间后续可扩展为 `password-protected`，不阻塞当前账号托管方案。

## Access Modes

```ts
type HomeSpaceAccessMode =
  | "account-managed"
  | "sync-code"
  | "password-protected";
```

- `account-managed`：账号可信托管模式。账号保存恢复凭证，用户不直接看到完整同步码；有效用户首页可进入账号托管云端历史，用于恢复、完整预览和审计。
- `sync-code`：普通同步码模式。用户持有完整同步码；不登录也能同步；空白设备必须输入同步码，云端默认只保存密文。
- `password-protected`：未来高隐私模式。账号可以看到空间元数据，但展示内容需要额外密码或恢复短语。

## Account-managed Home Space

账号托管首页空间是登录用户的默认模式。

用户价值：

- 登录后创建首页空间，不需要理解同步码。
- 空白设备登录账号后，可以看到并恢复全部账号托管空间。
- 首页编辑、同步、删除、默认空间和空间切换都收束到账号下的首页空间。

实现方向：

- 创建账号托管空间时，系统自动生成访问凭证。
- 当前已实现阶段仍以前端通过本人 RLS 读取托管凭证、再拉取和解密首页内容；Phase 1.11.5 已新增账号托管空间可恢复、可预览、可审计的云端历史记录。
- 账号托管凭证保存到账号可读的凭证表，通过 RLS 限制 `user_id = auth.uid()`；用户界面不展示完整同步码，也不能读取其他用户凭证。
- “前端完全不接触 managed secret”的服务端托管模型留到后续受控后台或服务端 RPC 阶段。
- 设置页默认展示首页空间，不直接展示完整同步码。
- 同步码管理入口逐步移动到“高级 / 恢复 / 同步码”区域。

## Sync-code Home Space

同步码空间继续作为匿名和兼容模式。

用户价值：

- 不注册账号也能跨设备同步。
- 已有用户仍可继续使用当前同步码。
- 可以作为临时恢复、设备迁移和高级分享入口。

边界：

- 只有完整同步码持有者能读写对应空间。
- 账号只认领此类空间时，不保存 `accessToken`、`encryptionKey` 或完整同步码。
- 空白设备登录账号后，只能看到被认领的空间索引；如果没有托管凭证，仍需输入完整同步码。

## Claim vs Migration

已绑定同步码的用户登录账号后，应提供两个动作。

### 仅认领同步码空间

账号只保存空间索引：

- `user_id`
- `sync_space_id`
- `name`
- 状态和时间元数据

效果：

- 账号能看到此空间。
- 当前设备继续用本地同步码凭证同步。
- 空白设备登录账号后不能直接恢复内容，必须输入完整同步码。
- 适合不希望账号托管凭证的用户。

### 迁移为账号托管空间

客户端使用当前同步码解密首页文档，然后迁移到账号托管空间。

流程：

1. 当前浏览器必须已绑定完整同步码。
2. 客户端拉取并解密当前首页文档。
3. 系统创建新的 `account-managed` 首页空间。
4. 系统生成账号托管凭证。
5. 使用账号托管凭证重新加密并上传首页文档。
6. 新空间写入账号下的 `home_spaces`。
7. 提示用户是否废弃旧同步码空间。

效果：

- 后续用户不需要同步码。
- 空白设备登录账号即可恢复此首页空间。
- 原同步码可以废弃，降低泄露风险。

## Data Model Direction

当前 `sync_spaces` 可继续作为底层密文文档表，但需要新增账号托管凭证能力。

建议新增：

```text
home_spaces
  access_mode
  credential_id

home_space_credentials
  id
  home_space_id
  user_id
  credential_type
  access_token
  encryption_key
  created_at
  revoked_at
```

说明：

- `home_spaces.access_mode` 区分 `account-managed`、`sync-code` 和未来 `password-protected`。
- `home_space_credentials` 只对所属账号可读。
- 字段命名可进一步改为 `managed_access_token`、`managed_encryption_key`，避免和用户可见同步码混淆。
- 当前阶段接受账号托管凭证由 RLS 保护，不要求零知识加密，也不把账号托管空间描述为严格端到端加密。
- 未来 `password-protected` 可将凭证改为用户密码或恢复短语加密后的密文。

## Security Requirements

- 用户只能读取自己的首页空间和托管凭证。
- `anon` 不能读取任何账号托管凭证。
- 前端只使用 Supabase anon key 和 RLS，不使用 service role。
- 托管凭证不得出现在 URL、公开页面、错误信息、日志或账号空间列表中。
- 普通同步码空间继续以密文保存，数据库不保存可直接查看的首页明文，也不显示可预览云端历史。
- 账号托管空间在 Phase 1.11.5 已开始保存可审计 `document_json` 云端历史；Phase 1.11.6 已明确这是账号可信托管模型，不是严格端到端加密。后续后台查看仍必须通过 RLS、受控后台接口和管理员审计限制访问。
- 从账号移除空间、迁移空间、废弃旧同步码必须二次确认。
- 账号托管空间的主要风险是账号被盗；高隐私需求进入 `password-protected` 模式。

## Updated Roadmap Integration

### Phase 1.5

已完成账号资料、空间索引、同步码认领、空间切换和安全回归。

- Phase 1.5.5 已按当前同步码模型完成空间切换。
- 如果当前设备没有某空间完整同步码，仍要求用户输入完整同步码。
- 本阶段未引入账号托管凭证表，避免扩大安全和迁移范围。

### Phase 1.6

新增账号托管同步作为重点。

- Phase 1.6.0 已新增 `006_account_managed_sync_foundation.sql` 草案，包含 `home_spaces.access_mode`、`home_space_credentials`、RLS 和 `create_account_managed_home_space(...)` RPC。
- Phase 1.6.1 已接入前端创建 `account-managed` 空间，创建成功后自动绑定当前浏览器。
- Phase 1.6.1a 已补齐恢复默认同步保护：已绑定同步码或账号托管空间时，清空恢复默认会进入 `paused` 状态，暂停自动上传/拉取；用户可选择上传默认、拉取云端、解除本机或恢复备份，避免误以为重新登录会自动恢复云端旧内容。
- Phase 1.6.2 已实现空白设备登录账号后直接恢复账号托管空间：前端读取本人 `home_space_credentials`，拉取并解密账号托管空间，再保存当前浏览器绑定。
- Phase 1.6.3 已实现已认领同步码空间“原地迁移为账号托管”；迁移后账号保存托管凭证，旧同步码本阶段仍保留有效。
- Phase 1.6.4 已实现账号首页空间 CRUD 收口：空间可重命名、设默认、从账号移除；移除账号托管空间会删除账号侧托管凭证，但不删除或废弃底层 `sync_spaces`。
- Phase 1.6.4a 已补齐删除策略收口：UI 统一使用“从账号移除”，当前本机账号托管空间禁止移除，并新增删除策略验证脚本。
- Phase 1.6.5 已将登录用户的同步码入口降级到离线同步码与恢复区域：`首页空间` 成为账号用户主入口，未登录同步码主入口保持不变，`paused` 和 `conflict` 处理不被折叠隐藏。
- Phase 1.6.5a 已补强同步码管理边界：离线同步码区域会区分当前绑定是否已属于账号空间，并明确 `解除本机`、`废弃同步码`、`从账号移除` 和账号托管凭证的不同影响。
- Phase 1.6.6 已补齐全局偏好编辑：账号偏好和未登录本地偏好并行，主题、字体、密度、日期格式和默认搜索引擎不写入首页文档，也不触发同步码上传。
- “为空间生成新同步码”已降级为 P4/未定义：该能力对账号托管主路径可用性提升有限，且会增加同步码生命周期、旧设备兼容和凭证边界复杂度。

### Phase 1.7

Phase 1.7 不再作为账号托管凭证治理阶段，改为专注组件开发。账号托管同步在本阶段只做必要回归，确保 Todo、日历等 widgets 写入 `HomeDocumentV2` 后仍能通过账号托管空间和同步码正常同步。

Phase 1.7.0 已接入 Widget registry 和组件面板增删排序，组件仍随完整首页文档保存与同步，不新增账号托管凭证字段或后端表。

Phase 1.7.1 已接入 Todo List v1，任务数据内嵌在 `todo.list` widget config 中，继续随完整首页文档加密同步；本阶段不新增任务表、不暴露 Todo 明文给 Supabase。

Phase 1.7.2 已接入日历/万年历 v1，周起始配置内嵌在 `calendar.month` widget config 中，继续随完整首页文档加密同步；翻月状态只保留在本地 UI，不新增日程表或外部日历授权。

Phase 1.7.3 已接入组件布局与编辑体验，组件标题、顺序和折叠状态继续内嵌在 `HomeDocumentV2.widgets` 中并随完整首页文档加密同步；管理模式、组件 picker 展开状态和拖拽临时态只保留在当前浏览器 UI。

Phase 1.7.4 已接入组件默认配置，模板默认组件在创建首页时生成普通 `HomeDocumentV2.widgets` 并随完整首页文档加密同步；本阶段不新增账号托管默认组件偏好，也不改动已有首页。

### Phase 1.8

Phase 1.8 聚焦主题和普通级个性化。Phase 1.8.0 已接入空间级主题 preset，主题字段继续内嵌在 `HomeDocumentV2.theme` 中并随完整首页文档加密同步；账号托管同步在后续 Banner、背景图片阶段需要继续验证图片字段不会泄露 secret，也不会破坏空白设备账号恢复。

### Phase 1.9

Phase 1.9 先做前端页面布局和 UI/UX 优化，再进入浏览器收藏/标签导入独立需求集。导入需求与账号托管同步只保持数据兼容要求：导入结果最终仍应进入普通 `HomeDocumentV2`，不得引入同步码 secret 泄露路径。具体导入方案暂不细化。

### Phase 1.10

正式推出前再集中做账号治理和优化收口。

- 一个首页空间支持多个设备凭证的可行性评估。
- 支持撤销单个设备或单个凭证的可行性评估。
- 多同步码和空间新同步码保持 P4/未定义，除非后续有明确高价值使用场景。
- 操作审计记录创建空间、迁移、废弃同步码、撤销设备和切换空间。
- `password-protected` 空间和只读分享链接先进入安全设计，不直接进入近期实现。

### Phase 1.11

数据保全与恢复体系成为 Phase 1.11 主线。账号托管同步需要配合完成：

- 文档分类：默认页、空白页和未编辑模板页不作为有效用户数据；用户编辑后的模板页和正常编辑首页合并为有效用户首页。
- 完整快照：快照、恢复和同步范围是完整 `HomeDocumentV2`，包含网站、主题、Banner/背景、组件、布局和同步状态。
- 云端历史版本：Phase 1.11.5 已实现账号托管空间保存可恢复、可预览、可审计的历史版本；普通同步码空间不保存明文历史。
- 恢复模型收口：Phase 1.11.6 已统一账号托管、普通同步码和未来高隐私模式的产品文案、设置页提示、数据包导出诊断和只读权限验证脚本。
- 危险写入保护：恢复默认、导入、模板、空间切换和云端覆盖前先保存有效用户首页快照。
- 后台审计：管理员 dashboard 可查看非离线加密数据和操作记录，但访问本身必须写入审计日志。

### Phase 1.12

组件设计和优化阶段只做账号托管同步必要回归，确保组件配置、排序、折叠、模板默认组件和移动端布局优化仍然写入普通 `HomeDocumentV2.widgets`，不破坏 Phase 1.11 的数据保全边界。

### Phase 1.13

主域名准备和 Auth redirect 回归延期到 Phase 1.13，和自购域名、`basePath`、静态资源路径、本地缓存域隔离一起集中处理。

### Phase 2+

扩展 AI 能力、高级定制化、商业化和更复杂的访问模型。

- AI 能力作为 Phase 2 核心功能，具体需求细节待定；涉及首页内容、链接、组件数据或用户偏好时，需要明确数据授权和隐私边界。
- WebGL 动态/交互背景和付费高级组件。
- 公开主页。
- 自定义域名绑定到 `home_space`。

## Acceptance Criteria

- 登录用户创建首页空间后，不需要看到同步码也能同步。
- 空白设备登录账号后，能看到账号下所有 `account-managed` 空间。
- 空白设备选择 `account-managed` 空间后，能拉取、解密并展示首页。
- 未登录用户仍可通过同步码同步。
- 已绑定同步码的登录用户可以选择仅认领或迁移为账号托管。
- 仅认领不会保存同步码 secret。
- 迁移为账号托管后，账号可直接恢复该空间内容。
- 用户 A 不能读取用户 B 的空间或托管凭证。
- 普通同步码空间数据库不保存首页明文；账号托管空间已在 Phase 1.11.5 保存可审计明文云端历史，必须只允许本人通过 RLS 正常读取和管理员通过受控后台审计访问。
